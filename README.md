# MultiPhysics-DGS-CA Streamlit App

This repository contains a Streamlit GUI for predicting the eccentric load-strain curve of concrete-filled steel tubular members using the **MultiPhysics-DGS-CA** model.

## Repository structure

```text
.
├── streamlitapp.py
├── requirements.txt
├── README.md
├── .gitignore
├── .streamlit/
│   └── config.toml
└── models/
    └── ea_pgcc_dgs_ca_model_package.pkl
```

## Model file

The app expects the trained model package at:

```text
models/ea_pgcc_dgs_ca_model_package.pkl
```

This file is generated by the training script and contains:

- the trained Dynamic Gated Stacking model;
- CatBoost-anchor blending parameters;
- physics-guided parameters alpha, beta, gamma;
- raw feature columns;
- model feature columns;
- categorical mappings;
- conformal prediction interval parameter, if available.

If the model package is too large for GitHub, do not upload it directly. You can either:

1. manually upload it inside the Streamlit app; or
2. host it on Google Drive / Hugging Face / GitHub Releases and set `MODEL_PACKAGE_GDRIVE_ID` in `streamlitapp.py`.

## Run locally

```bash
pip install -r requirements.txt
streamlit run streamlitapp.py
```

## Deploy on Streamlit Community Cloud

1. Upload this repository to GitHub.
2. Go to Streamlit Community Cloud.
3. Select this repository.
4. Set the main file path to:

```text
streamlitapp.py
```

5. Deploy.

## Input parameters

The model uses the following raw features:

```text
D/t
L/D
Helix Angle
fcuAc
fyAs
h/l
Concrete Types
e/r
Confinement Factor
Strain
```

The GUI asks for engineering parameters such as `D`, `t`, `L`, `h`, `l`, `fcu`, `fy`, eccentricity, and strain range, then automatically constructs the model input features.

## Prediction formula

The app reconstructs the internal physics-guided features and predicts:

```text
N0_Enhanced = (fcuAc * (1 + alpha * xi) + fyAs) / 1000
PhysicsReduction = 1 / (1 + beta * (e/r) + gamma * (L/D))
N_phy = N0_Enhanced * PhysicsReduction
R_final = anchor_alpha * R_DGS + (1 - anchor_alpha) * R_CatBoost
Nu_pred = BoundaryFactor * N_phy * R_final
```

## Notes

- `N0_Enhanced`, `PhysicsReduction`, `N_phy`, and `BoundaryFactor` are internal derived features and are not manual GUI inputs.
- Keep the input units consistent with the training database.
- If deployment fails at `joblib.load()`, check the Python and package versions, especially `scikit-learn`, `xgboost`, `lightgbm`, and `catboost`.
